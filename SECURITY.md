# Security Policy

## Supported Versions

Поддерживается актуальное состояние ветки `main` (best-effort).
Backport исправлений в прошлые состояния репозитория не гарантируется.

## Reporting a Vulnerability

- Не публикуйте детали уязвимости в публичных issues.
- Используйте private security advisory в GitHub-репозитории.
- В отчёте укажите:
  - краткое описание уязвимости;
  - шаги воспроизведения;
  - оценку потенциального влияния;
  - предложенный mitigation (если есть).

Отчёты рассматриваются в best-effort режиме, без формальных SLA по срокам ответа и релиза исправления.
