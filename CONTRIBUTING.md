# Contributing Guide

Спасибо за интерес к проекту.

## Как предложить изменение

1. Создайте issue с описанием проблемы или идеи.
2. Сделайте форк и создайте ветку от `main` (например, `feature/new-forecast-metric`).
3. Внесите изменения небольшими логическими коммитами.
4. Проверьте, что тесты проходят локально.
5. Откройте Pull Request и заполните шаблон.

## Локальная разработка

```bash
python -m venv .venv
.venv\Scripts\activate
pip install -r requirements.txt -r requirements-dev.txt
pytest -q
```

## Правила для Pull Request

- один PR - одна задача;
- описывайте, что и зачем изменено;
- добавляйте/обновляйте тесты при изменении поведения;
- не включайте в PR бинарные артефакты (`*.joblib`) и локальные отчёты.

## Code Style

- следуйте текущему стилю проекта;
- пишите понятные имена функций/переменных;
- избегайте "магических" чисел без объяснения.

## Отчёты о безопасности

Пожалуйста, не публикуйте уязвимости в открытых issues. Используйте процесс из `SECURITY.md`.
